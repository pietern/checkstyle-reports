7   /**
8    * This interface should be implemented by all Infinispan modules that expect to have components using {@link Inject},
9    * {@link Start} or {@link Stop} annotations.  The metadata file is generated at build time and packaged in the module's
10   * corresponding jar file (see Infinispan's <pre>core</pre> module <pre>pom.xml</pre> for an example of this).
11   * <p />
12   * Module component metadata is usually generated in a file titled <pre>${module-name}-component-metadata.dat</pre> and
13   * typically resides in the root of the module's jar file.
14   * <p />
15   * For example, Infinispan's Query Module would implement this interface to return <pre>infinispan-query-component-metadata.dat</pre>.
16   * <p />
17   * Implementations of this interface are discovered using the JDK's {@link java.util.ServiceLoader} utility.  Which means
18   * modules would also have to package a file called <pre>org.infinispan.factories.components.ModuleMetadataFileFinder</pre>
19   * in the <pre>META-INF/services/</pre> folder in their jar, and this file would contain the fully qualified class name
20   * of the module's implementation of this interface.
21   * <p />
22   * Please see Infinispan's query module for an example of this.
23   *
24   * @author Manik Surtani
25   * @since 5.1
26   */
27  public interface ModuleMetadataFileFinder {
28     String getMetadataFilename();
29  }
